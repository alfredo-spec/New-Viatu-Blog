1. OVERVIEW
───────────────────────────────────────────────────────────────
This folder contains design mockups for a dedicated long-form
blog details page. Unlike the existing /blog-post/ mockups
(which model lodge review articles with rating boxes), these
target editorial guide content: 15+ min reads with multiple
sections, image galleries, timelines, FAQs, and CTAs.

Three full-page variants explore different design philosophies:

  v1-classic-editorial   Book-like, minimal, readability-first
  v2-magazine-bold       Bold typography, section color rhythm
  v3-immersive-cinematic Photography-driven, dark sections, drama

4. SHARED DATA STRUCTURE
───────────────────────────────────────────────────────────────

All three variants use the same underlying content model:

  Article {
    title:        string    // H1 headline
    subtitle?:    string    // Lead text (v3 uses in hero)
    author: {
      name:       string    // "Barbara Buchel"
      role:       string    // "Safari Specialist & Editor"
      avatar:     string    // image path
      bio:        string    // 2-3 sentences
    }
    published:    string    // "October 2024"
    readingTime:  string    // "15 min read"
    category:     string    // "Safari Guide" (eyebrow)
    tags:         string[]  // ["Tanzania","Safari","Wildlife Guide","Great Migration"]
    breadcrumbs:  string[]  // ["Blog","Destinations","Tanzania"]
    heroImage:    string    // image path
    sections:     Section[] // ordered content blocks
    faqs:         FAQ[]     // question/answer pairs
    relatedPosts: Card[]    // 3 related article cards
  }

  Section {
    id:           string    // anchor ID for TOC
    tagline?:     string    // Caveat tagline (v2 only)
    heading:      string    // H2 text
    content:      Block[]   // mixed content blocks
  }

  Block = TextBlock | ImageBlock | GalleryBlock | PullQuoteBlock
        | CalloutBlock | TimelineBlock | ListBlock | CTABlock

  TimelineEntry {
    month:        string    // "January" or "Feb – Mar"
    location:     string    // "Southern Serengeti"
    description:  string    // brief text
    season:       'calving' | 'breeding' | 'crossings' | 'return'
    image?:       string    // photo path (v3 only)
  }

───────────────────────────────────────────────────────────────

  768-1024    Tablet: TOC collapses, grids reduce columns
  480-768     Mobile: single column, stacked layouts
  < 480       Small mobile: reduced padding, smaller type


8. JAVASCRIPT PATTERNS
───────────────────────────────────────────────────────────────

Shared across all variants:
  - Nav scroll toggle (.scrolled at scrollY > 80)
  - Language dropdown (click, outside-click, escape)
  - Reveal on scroll (IntersectionObserver, .vis class)
  - FAQ accordion (click toggle, single-open)

Variant-specific:
  v1: TOC scroll spy (sidebar), mobile TOC toggle
  v2: TOC scroll spy (pill bar), auto-scroll active pill,
      reading progress bar (gradient)
  v3: Parallax hero, floating TOC toggle + panel,
      SVG circle progress, overlay close handlers

TypeScript interfaces:

  interface BlogArticle {
    title: string
    subtitle?: string
    author: Author
    published: string
    readingTime: string
    category: string
    tags: string[]

  interface Author {

    | { type: 'timeline'; entries: TimelineEntry[] }

  }

  interface FaqItem {

    answer: string
  }

  interface RelatedPost {
    title: string
    excerpt: string
    category: string
    author: string
    date: string
    image: string
    url: string
  }

11. DESIGN DECISIONS & RATIONALE
───────────────────────────────────────────────────────────────

Heading hierarchy fixed:
  The original blog uses H1 → H3 (skipping H2). All variants
  correct this to H1 → H2 → H3 for semantic HTML.

Reading progress:
  At 15 min / 4000+ words, a progress indicator reduces reader
  overwhelm and encourages completion. v1/v2 use a subtle top
  bar; v3 uses a floating circle for cinematic cleanliness.

Table of contents:
  Three approaches tested: sidebar (traditional), pill bar
  (modern/scrollable), floating panel (minimal). All include
  scroll-spy to track reading position.

Section color alternation (v2):
  Only used for "feature" sections (galleries, timeline, FAQ) —
  not every section. This signals content-type changes without
  disrupting reading flow.

Dark sections (v3):
  Used sparingly for maximum impact: galleries, timeline, pull
  quotes, author bio, newsletter, related posts. The contrast
  between white reading sections and dark visual sections creates
  a cinema-like experience.

Month-by-month timeline:
  Replaces the original wall-of-text month paragraphs with
  scannable visual formats. Three approaches: simple vertical
  (v1), scrollable cards (v2), photo-rich horizontal (v3).

Mid-article CTA:
  Added at ~60% of the article to catch readers who won't reach
  the bottom. Uses distinctly different styling per variant so it
  doesn't feel intrusive.

Pull quotes:
  Three styles offer different levels of visual interruption.
  v1 is subtle (left border), v2 is decorative (centered), v3
  is dramatic (full-width dark section).
