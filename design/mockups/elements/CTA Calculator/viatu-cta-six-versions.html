Blog CTA components

Six versions — three layouts, two moods

Component Overview

Each CTA card is a self-contained component designed to sit inside blog post templates. The component accepts data-attribute toggles on the root .card element to control which sections are visible — allowing editorial flexibility without code changes.

Configuration Toggles

Add these data-* attributes to the root .card element. All default to "true" when omitted.

AttributeTypeDescription
data-show-tagline"true" | "false"Caveat tagline above the title (e.g. "Interested in this trip?")
data-show-subtitle"true" | "false"Subtitle text below the title (e.g. pricing hint)
data-show-consultant"true" | "false"Consultant badge on the image panel (name, role, quote)
data-show-phone"true" | "false"Phone number field (with country code search dropdown)
data-show-budget"true" | "false"Budget slider + estimated total output block
data-show-duration"true" | "false"Duration slider
data-show-travellers"true" | "false"Adults + children dropdowns
data-show-total"true" | "false"Estimated total output (auto-hides if budget is hidden)
data-show-usps"true" | "false"Trust pills at the bottom (24h, no obligation, B Corp)
Example — minimal form:
<div class="card mu" data-show-phone="false" data-show-budget="false" data-show-duration="false" data-show-travellers="false" data-show-total="false">
This renders only: name, email, and the submit button — a lightweight lead capture form.

Dynamic Content Slots

SlotAcceptsNotes

Layout Variants

TypeClassBehaviour
Version A — WizardUses .steps + .pnl panelsStep 1: contact → Step 2: trip config → animated confirmation. JS wNext() / wBack() / wConfirm() controls transitions.
Version B — Single formUses .fsec section dividersAll fields visible in one scroll. Sections separated by labelled dividers. Submit triggers sConfirm() to replace form with confirmation.
Version C — Trip-first wizardUses .steps + .pnl panels (reversed)Step 1: trip config → Step 2: contact → animated confirmation. Same wizard JS, reversed content order. Lets users engage with the fun part first.

Phone Prefix Component

The phone prefix uses a custom searchable dropdown (.phone-prefix). It supports keyboard navigation, search-as-you-type, and click-outside-to-close. The country list is defined in the JS COUNTRIES array — add or remove entries there.

Accessibility: The dropdown uses role="listbox", aria-expanded, and keyboard handlers for arrow keys + Enter + Escape. The search input gets focus when the dropdown opens.

Budget Calculator

Formula: budget_per_person_per_day × duration_days × (adults + children). The total updates in real time via the cT(prefix) function. Each variant uses a two-letter prefix (AM, AL, BM, BL) to namespace its IDs.

Responsive Behaviour

Below 768px, the card stacks vertically (image panel on top, form below). The image panel collapses to 180–220px height. Two-column field rows become single-column. The budget output centers vertically.
